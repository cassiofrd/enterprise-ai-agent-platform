# Evaluation module

This folder contains a lightweight evaluation workflow for the supply-chain agents.

The goal is to make agent quality measurable and repeatable. Instead of manually
asking questions in the UI, this module runs a CSV of test questions against an
agent endpoint and writes the answers to a results CSV.

## Folder structure

```text
evaluation/
├── datasets/
│   └── inventory_agent_eval.csv
├── results/
├── reports/
├── run_agent_eval.py
└── compare_runs.py
```

## Dataset format

The input dataset must have at least:

```csv
query,ground_truth,category
"Qual o fornecedor do PARAFUSO-M20?","O fornecedor preferencial do PARAFUSO-M20 é XYZ Metais.","valid_direct"
```

Required columns:

- `query`: question sent to the agent
- `ground_truth`: expected answer or expected refusal

Optional columns:

- `category`: useful for breaking down pass rate by test type

## Run against local Supervisor Agent

Start the local services first. Then run:

```bash
python evaluation/run_agent_eval.py \
  --agent-url http://localhost:8000/copilot \
  --agent-type supervisor_copilot
```

## Run against local Inventory Agent directly

```bash
python evaluation/run_agent_eval.py \
  --agent-url http://localhost:8001/invoke \
  --agent-type inventory_invoke
```

## Run against Azure Container Apps

Use the deployed endpoint:

```bash
python evaluation/run_agent_eval.py \
  --agent-url https://<your-supervisor-container-app-url>/copilot \
  --agent-type supervisor_copilot
```

You can also configure through environment variables:

```bash
export EVAL_AGENT_URL="https://<your-supervisor-container-app-url>/copilot"
export EVAL_AGENT_TYPE="supervisor_copilot"
python evaluation/run_agent_eval.py
```

## Compare or summarize runs

```bash
python evaluation/compare_runs.py evaluation/results/inventory_agent_results_*.csv --show-failures
```

## Uploading results to Azure AI Foundry

The generated results CSV contains:

```csv
query,response,ground_truth
```

That format can be used in Azure AI Foundry dataset evaluation flows, because
the candidate `response` has already been generated by the agent.

## Notes

The `contains_ground_truth` column is a simple deterministic smoke-test metric.
It is not a replacement for Foundry evaluators such as Relevance, Fluency,
Coherence, Quality Grader, Groundedness, or custom LLM-as-judge evaluation.
